Tin Cup 2026 Admin Guide

This page documents the current Tin Cup admin workflow: where data is entered, which items are manual, and which pages are verification-only.

Page Map

Tin Cup 2026 Main trip page. Use it to review the schedule, open Trip Info, jump to Scoring / Inputs, Live Results, Lodging, and add trip tee times to your calendar.
Scoring / Inputs Main admin workspace. This is where live scores, scramble scores, markers, side-game winners, golfer setup, handicap settings, penalties, payout values, and live-access toggles are managed.
Live Results Read-only verification page. Use it to confirm totals, match points, scramble points, skins, side games, payouts, and audit rows after entries are saved.
Lodging Manual condo, room, extra-night, and payment management. The golfer list on this page comes from the Scoring / Inputs roster and should not be edited here.

Main Trip Page

  • Use Trip Info to open the travel and trip details modal.
  • Use the calendar to switch days inside the current Tin Cup trip month. The calendar is locked to the trip month and does not move outside it.
  • Use Add trip tee times to your calendar to download the trip tee times into a calendar app.
  • Use the top chips to open Live Results, Lodging, Scoring / Inputs, and this guide.
  • Event cards, tee times, travel links, map links, and course details are primarily review surfaces on this page, not the main place for scoring changes.

Live Scorecard Access

  • Choose a round from the Day selector. This includes Scramble in the same selector as the other rounds.
  • Open a group link to launch the live scorecard for a foursome or scramble team.
  • Scorers enter their name when opening a card. If code protection is enabled, they must also enter the group code.
  • Use the hole jump strip to move between holes. The mobile score pad supports -, +, Clear, and quick score chips.
  • Use Full Scorecard only for review. Hole-by-hole entry is the main scoring surface.
  • On non-scramble rounds, the scorecard auto-locks as soon as all 18 holes are entered for every player. Editing a submitted card still requires an admin-code override.

Markers And Side Games

  • Live Markers is where Closest To Pin and Long Drive winners are assigned on applicable holes.
  • Side Game Winners is where Long Putt is assigned manually and Secret Snowman is drawn from a random score of 8 once a full day is entered.
  • The leaderboard and payout tables update from the Long Putt selection and the Secret Snowman draw result saved there.
  • If marker entry is hidden, re-enable it from the Admin Tools live-scoring toggles.

Golfer Roster Setup

  • Use Add Golfer Row to insert a blank player row.
  • Use Import Golfers CSV to load roster rows from a CSV file.
  • Use Save Golfer Setup after roster edits.
  • Each row includes Name, Handicap, Classification, and Penalty.
  • Classification is where previous champions and rookies are identified. Penalty is where manual scoring penalties are assigned to a golfer.
  • This roster is the source for the Lodging page. Do not try to add golfers from Lodging.

Scoring Rules And Payout Config

These values drive handicap conversion, points, side-game payouts, scramble points, and the final payout model.

Handicap fields Index->HC %, Avg Course Slope, Avg Slope, Max HC, and Par 3 Fraction.
Trip pot fields Entry Fee, Winner, 2nd, 3rd, and 4th.
Side-game payouts Long Drive, CTP, Long Putt, Secret Snowman, Loser, and Skins / Person.
Scramble points Scramble 1st, Scramble 2nd, and Scramble 3rd.
Save action Use Save Scoring Config after changes. The surrounding Pot Summary and handicap verification table are there to confirm the numbers before and after saving.

Admin Tools

  • Live Foursome Scoring: turns access to the Scoring / Inputs live entry flow on or off.
  • Closest To Pin / Long Drive Entry: shows or hides marker entry on the scoring page.
  • Live Leaderboard Page: shows or hides the dedicated live results page and nav chip.
  • Code-Protected Group Scorecard Links: requires each group code before a live scorecard opens.
  • Admin Code is used for protected admin actions, including code generation, demo score seeding, and submitted-card override edits.
  • Generate Missing Codes creates any missing foursome codes.
  • Seed All Rounds Demo Scores loads demo winners and results for testing every side game and scoring surface.
  • Clear Competition removes seeded scorecards, scramble scores, markers, side-game winners, and penalties so Tin Cup is ready for Round 1 while keeping player names, handicaps, live settings, foursome codes, and scoring config.
  • Export Final Competition CSV becomes available after the trip end date and downloads a full archive of final standings, scorecards, scramble results, markers, skins, side games, and payouts.

Lodging Page

  • Use the status summary and condo groups to see who is assigned, unassigned, or outside the active condo count.
  • Manual lodging inputs include Assign to condo, Move condo, Room, Unassign, Extra night, and Paid.
  • The Lodging page is also where Tin Cup payment amounts are maintained.
  • The roster list itself is not edited here. If a golfer is missing or needs to be removed, do that in Scoring / Inputs.

Live Results Page

  • Use Day to switch the live day leaderboard.
  • Use the match day tabs to review hole-by-hole match results by day.
  • Review read-only sections for stroke results, skins, scramble, side games, payouts, score rankings, and results audit.
  • If numbers look wrong here, the fix normally belongs on the Scoring / Inputs or Lodging page, not on the leaderboard page itself.

Automatic Vs Manual

Automatic calculations: trip totals, day totals, handicap stroke application, penalties, skins, scramble points, payout rollups, and audit tables after data is saved.
  • Manual entry surfaces: roster rows, handicaps, classifications, penalties, hole scores, markers, Long Putt, Secret Snowman, live-access toggles, admin codes, condo assignments, extra nights, and payment amounts.
  • Verification surfaces: Live Results, Pot Summary, handicap verification, full scorecard review, and trip-page schedule/event displays.

Recommended Admin Workflow

  1. Before the trip, confirm the roster, handicap values, classifications, penalties, payout config, scramble points, and lodging assignments.
  2. Before each round, confirm the correct day is selected, live scoring is enabled, and group codes are generated if code protection is on.
  3. During play, enter hole scores first, then marker winners for applicable holes, then side-game winners when those results are known.
  4. For scramble day, enter team hole scores from the same live score-entry flow used by the other rounds.
  5. After a round, submit scorecards, review Live Results, and use the Results Audit section to confirm totals and payouts.
  6. If testing is needed, seed demo scores from Admin Tools and use the resulting summaries to verify every scoring surface.
