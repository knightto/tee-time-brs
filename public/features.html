Tee Time BRS Features

This page lists current product capabilities. For step-by-step instructions, use the User Guide. For routing, email, hosting, and scheduler details, use the Operations Guide.

Public Site Capabilities

  • Group-aware public pages: main BRS and Seniors run from the same app with different branding, manifests, permissions, contacts, and notification scope.
  • Installable mobile app behavior: Add to Home Screen support with group-specific manifests and standalone launch behavior.
  • Calendar navigation: month switching, date selection, selected-day tee sheet, and mobile bottom navigation.
  • Subscribe: group-scoped notification sign-up with unsubscribe links tied only to that group.
  • Monthly calendar export: one-click ICS download for the visible month.
  • Day and event sharing: share links and clipboard fallback for selected events and day summaries.
  • Calendar tee-time request: request a new club tee time for a selected date and time.
  • Course enrichment: city, state, phone, website, holes, par, and weather summary when available.
  • Fresh mobile behavior: automatic refresh on resume, focus, online recovery, and page restore.
  • Visual urgency: red empty tee-time markers and near-term empty-slot emphasis.
  • Starter mode: compact day-of board with filters, urgency counts, check-in shortcuts, and denser tee-sheet layout.
  • Audit visibility: event audit access from the public card UI where enabled.

Event And Tee Sheet Tools

  • Event types: tee-time events, team events, and Seniors event-only signup events.
  • Event actions: starter view, share, add time or group, suggest pairings, Google Calendar add, ICS export, and audit access where allowed.
  • Player actions: add, move, remove, individual check-in, and check-in-all workflows.
  • Drag and drop: supported on compact starter-style boards and trip practice-round layouts where enabled.
  • Maybe list: interested list management with fill-spot support.
  • Fifth-player handling: practice and special cases can allow a clearly marked 5th golfer in a foursome.
  • Weather-aware tee cards: event cards show forecast summaries directly in the header.
  • Empty-slot guidance: event cards highlight open and urgent empty tee times for operational follow-up.

Seniors-Specific Behavior

  • Public Seniors page: golfer-level changes only, with full event and calendar administration kept off the public page.
  • Seniors admin page: full event and tee-time editing after admin-page access without repeated item-level code prompts.
  • Roster-aware signup filtering: golfers already assigned to a Seniors event are hidden from add and sign-up pickers until removed.
  • Simple signup events: event-only registration mode without tee-time rows when needed.
  • Scoped notifications: Seniors subscribers and lifecycle notifications remain isolated from the main BRS group.

Trips And Competition Pages

  • Tin Cup: themed trip site with dedicated mobile manifest, trip schedule, score entry, leaderboard, lodging, winners mode, and practice-round management.
  • Myrtle Trip: separate trip pages and competition support.
  • Blue Ridge Outings: outing schedule and public reference page.
  • Handicaps and rules: supporting pages for handicaps and weekend game rules.

Admin And System Functions

  • Global notifications toggle: enable or disable broad subscriber mailouts.
  • Automated scheduler toggle: pause or resume scheduled jobs.
  • Scheduled rule controls: reminders, empty-tee alerts, nearly-full alerts, and group-specific operational notices.
  • Quick actions: refresh weather, verify course data, inspect empty tees, and send subscriber mailouts.
  • Subscriber management: review and remove subscribers by group.
  • Audit logs: adds, edits, deletes, moves, check-ins, and create entries where auditable records support them.
  • Email import: automatic event create and cancel handling from forwarded club emails.
  • Backups and deployment support: backup metadata, group deployment links, and operations references.
  • Regression coverage: server, frontend, mobile-shell, and Tin Cup browser tests are included in the repo scripts.
